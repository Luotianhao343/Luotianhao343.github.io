Spring是什么

  • Spring是一个开源框架，为了解决企业应用开发的复杂性而创建的，现在不止于应用于企业应用。
  • 是一个轻量级的控制反转和面向切面（AOP）的容器框架
    • 从大小和开销，Spring是轻量的
    • 通过IOC达到松耦合的目的
    • 提供AOP编程丰富支持，允许通过分离应用的业务逻辑与系统级服务进行内聚性的开发
    • 包含并管理应用对象的配置和生命周期，这个意义上是一种容器
    • 将简单的组件配置、组合成复杂的应用，这个意义上是框架

框架？

特点：

  • 半成品
  • 封装了特定的处理流程和控制逻辑
  • 成熟的、不断升级改进的软件

与类库的区别：

  • 框架一般是封装了逻辑、高内聚的，类库则是松散的工具组合
  • 框架专注某一领域，类库则是更通用的。（框架如楼房的钢管架构，类库则是钢材）

为什么使用框架？

  • 软件系统复杂
  • 重用度高，开发效率和质量提高
  • 易于上手

IOC

接口是什么？

类经过抽象和封装后，提供给外部一个能沟通使用的口；外部不需要知道内部实现，只需要提供本身的信息与接口对接便可完成操作。

所以接口的好处是：内部实现修改不会影响外部与接口对接的操作。

在Java8中，接口可以拥有方法体。

###面向接口编程

  • 结构设计中，分清层次和调用关系，每层只向外提供一组接口，各层之间依赖于接口而不是实现类（你提供信息，我提供接口方法；不用知道我如何操作，你拿到结果就行）
  • 接口内部实现的变动，不会影响各层之间的调用，在追求稳定的公共服务中比较重要
  • 面向接口编程中的接口是用于隐藏具体实现和实现多态性的组件。（多态：父类引用指向子类对象，并调用子类方法）

什么是IOC？

控制反转，将创建对象的控制权转移；应用程序本身不负责依赖对象的创建和维护，而是由外部容器负责创建和维护。（我买房，但是不创建房，由房地产商创建）

Spring Bean的装配

Bean的作用域

  • singleton:单例，一个Bean容器中只存在一份
  • prototype：每次请求（每次使用）创建新的实例，destory方式不生效

Bean的生命周期

  • 初始化：
    1. 实现InitializingBean接口，覆盖afterPropertiesSet方法
    2. 配置init-method
  • 销毁：
    1. 实现DisposableBean接口，覆盖destory方法
    2. 配置destory-method

初始化和销毁也可在xml全局默认中配置

如果三种初始化和销毁方式同时存在，容器加载的时候会先执行aferPropertiesSet方法再执行start，容器销毁的时候执行destory方法再执行stop方法。而全局默认配置的方法是默认不执行的。

Aware接口

Spring中提供了以Aware结尾的接口，实现了Aware接口的bean在被初始化后，用来获取容器中的Context信息。

  • BeanNameAware接口；实现setBeanName方法
  • ApplicationContextAware接口；实现setApplicationContext方法

XML与注解的优缺点：

  • 配置xml较多用于旧版本的spring，他更容易理解。缺点是需要配置很多xml文件，维护代码需要阅读xml文件
  • 越来越多的项目倾向于使用注解的方式，越少的配置越好
  • 在项目实施的时候，一个接口有多个实现类，对不同的用户，如果没有源代码，只需要修改配置文件就达到效果
